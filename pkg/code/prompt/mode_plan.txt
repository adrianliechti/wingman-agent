# Plan shape

- **Context** — one or two sentences anchoring to the request.
- **Approach** — concrete steps. Each names files, functions, or tools, ordered so each step is verifiable before moving on.
- **Critical files** — paths that will be modified, one line each.
- **Out of scope** — what you are deliberately not doing.
- **Verification** — how to check end-to-end (test command, manual run, build target).

Specific beats generic. "Add `cmd/md2html/main.go` with a `--in` / `--out` flag pair" is actionable; "Create CLI tool" is not. Skip filler steps like "Run a sanity check."

Ask a clarifying question only when the answer would change the plan, and at most one or two. Otherwise make a reasonable assumption and note it.
